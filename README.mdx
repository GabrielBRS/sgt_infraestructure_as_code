# Padrão de Endereçamento IPv6 SGT

Esta documentação define a taxonomia oficial de endereçamento IPv6 para as redes privadas da Sousa Global Technologies (SGT). A arquitetura foi desenhada para suportar microsegmentação, segurança Zero-Trust e escalabilidade de microsserviços e inteligência artificial.

## Estrutura do Subnet ID (Nibbles)

Utilizamos um prefixo ULA (Unique Local Address) base de `/48`. A organização lógica da rede ocorre nos 16 bits seguintes, divididos em 4 caracteres hexadecimais (N1 a N4), resultando em sub-redes `/64`.

**Formato:**
`[Prefixo Global /48] : [ N1 | N2 | N3 | N4 ] :: /64`

---

### N1: Ambiente (Ciclo de Vida)
Define o estágio de maturidade do tráfego.

| Hex | Ambiente | Descrição |
| :--- | :--- | :--- |
| `1` | **PROD** | Produção. Tráfego real, dados sensíveis e instâncias estáveis. |
| `2` | **STG** | Homologação (Staging). Espelho do ambiente de produção. |
| `3` | **DEV** | Desenvolvimento. Testes de novos artefatos e ambientes efêmeros. |
| `4` | **DR** | Disaster Recovery. Infraestrutura de backup e cold standby. |

---

### N2: Localidade (Infraestrutura Física)
Define a localização física ou o provedor do nó para roteamento macro.

| Hex | Localidade | Descrição |
| :--- | :--- | :--- |
| `1` | **On-Premises** | Servidores físicos locais (Bare-Metal). |
| `2` | **Public Cloud** | Provedor de nuvem principal de infraestrutura e serviços. |
| `3` | **Edge / Cloud Sec** | Provedores secundários ou nós de borda. |
| `4` | **VPN / Dial-up** | Redes de acesso remoto e túneis de desenvolvedores. |

---

### N3: Camada / Tier (Perfil de Tráfego e Função)
O núcleo das regras de firewall. Define a função do serviço e seu nível de isolamento.

| Hex | Camada | Tráfego e Função |
| :--- | :--- | :--- |
| `0` | **Gateway / Edge** | Recebe tráfego externo/internet (Proxies, Load Balancers). |
| `1` | **Frontend / UI** | Servidores de assets estáticos e renderização de interface. |
| `2` | **Backend Core** | APIs e camada de regras de negócios. |
| `3` | **Bancos de Dados** | Isolamento máximo (PostgreSQL, ScyllaDB, etc.). |
| `4` | **Mensageria** | Barramento de eventos (Kafka, RabbitMQ, etc.). |
| `5` | **Batch Processing** | Processos em background e workers genéricos. |
| `6` | **Machine Learning** | Inferência e treinamento de IA, nós de alta performance/GPU. |
| `7` | **Data Science** | Pipelines de ETL, workers de dados e ambientes de exploração. |
| `8` | **Cache / KV** | Armazenamento temporário de alta velocidade (Redis, Memcached). |
| `f` | **Observabilidade** | Monitoramento, telemetria (Prometheus) e gerenciamento (SSH). |

---

### N4: Cluster ID (Identificação do Nó/Zona)
Identifica zonas de disponibilidade ou clusters independentes da mesma camada.

| Hex | ID | Descrição |
| :--- | :--- | :--- |
| `0` | **Principal** | Cluster Primário / Zona A. |
| `1` | **Secundário** | Cluster Secundário / Zona B (Alta disponibilidade). |
| `2`-`e`| **Custom** | Segmentações arbitrárias específicas do serviço. |
| `f` | **Routing** | Tráfego interno de roteadores e gateways de interligação. |

---

## Exemplos de Topologia

Abaixo, simulações de geração de rede utilizando o prefixo de exemplo `fdb4:1029:8a7c::/48`:

```text
# 1. Gateway de Produção no Servidor Físico (On-Premises)
Prefixo: fdb4:1029:8a7c:1100::/64
(N1=1: Prod, N2=1: On-Prem, N3=0: Gateway, N4=0: Cluster Principal)

# 2. Cluster de Machine Learning de Produção (On-Premises)
Prefixo: fdb4:1029:8a7c:1160::/64
(N1=1: Prod, N2=1: On-Prem, N3=6: ML, N4=0: Cluster Principal)

# 3. Pipeline de Data Science em Desenvolvimento (Nuvem Pública)
Prefixo: fdb4:1029:8a7c:3270::/64
(N1=3: Dev, N2=2: Cloud, N3=7: Data Science, N4=0: Cluster Principal)

# 4. Banco de Dados Principal de Homologação (Nuvem Pública)
Prefixo: fdb4:1029:8a7c:2230::/64
(N1=2: Stg, N2=2: Cloud, N3=3: DB, N4=0: Cluster Principal)
