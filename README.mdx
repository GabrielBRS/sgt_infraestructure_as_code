# Padrão de Endereçamento IPv6 SGT

Esta documentação define a taxonomia oficial de endereçamento IPv6 privado (ULA - *Unique Local Address*) para as redes da Sousa Global Technologies (SGT). A arquitetura é construída para isolamento estrito de *tenants* (clientes) e microsegmentação de microsserviços.

## 1. O Prefixo Global /48 (Tenancy e Isolamento de Cliente)

Conforme a RFC 4193, endereços ULA utilizam o bloco `fd00::/8`, seguido por um **Global ID de 40 bits** gerado de forma pseudoaleatória.
Na SGT, **cada Global ID representa um Tenant (Cliente) ou uma Unidade de Negócio isolada.**

*   **Regra de Ouro:** Nunca crie um Global ID sequencial ou estético. Use algoritmos de hash para gerar o `/48` para garantir que a rede da SGT nunca colida com a rede de um cliente em fechamentos de VPN (site-to-site).

**Exemplo de Tabela de Tenants:**
*   `fd9a:3b21:8c77::/48` -> Tenant: SGT (Infraestrutura Interna)
*   `fde4:99a1:b2c3::/48` -> Tenant: Plataforma Solaria (HealthTech)
*   `fd11:aa22:bb33::/48` -> Tenant: Cliente Externo A

## 2. Estrutura do Subnet ID (Nibbles N1 a N4)

Dentro do `/48` do Tenant, organizamos os 16 bits seguintes em 4 caracteres hexadecimais, definindo as políticas de Zero-Trust e roteamento.

**Formato:**
`[Prefixo do Tenant /48] : [ N1 | N2 | N3 | N4 ] :: /64`

### N1: Ambiente (Ciclo de Vida)
Define a política de isolamento duro. Um ambiente nunca se comunica diretamente com outro.

| Hex | Ambiente | Descrição |
| :--- | :--- | :--- |
| `1` | **PROD** | Produção. Tráfego real e dados sensíveis. |
| `2` | **STG** | Homologação (Staging). Espelho de produção. |
| `3` | **DEV** | Desenvolvimento. |
| `4` | **DR** | Disaster Recovery / Cold Standby. |

### N2: Localidade (Topologia Física)
Discrimina a localização do processamento. *(Nota: Devido ao N1 ser mais significativo, este nível define escopo físico, mas não serve para sumarização BGP de data centers inteiros).*

| Hex | Localidade | Descrição |
| :--- | :--- | :--- |
| `1` | **On-Premises** | Servidores físicos locais (Bare-Metal). |
| `2` | **Public Cloud** | Provedor de nuvem principal (AWS, GCP, etc). |
| `3` | **POP Secundário** | Provedores de apoio, VPS ou Edge nodes. |
| `4` | **VPN / Dial-up** | Redes de acesso remoto. |

### N3: Camada / Tier (Perfil de Tráfego e Firewall)
Define a função do serviço e a matriz de comunicação permitida.

| Hex | Camada | Tráfego e Função |
| :--- | :--- | :--- |
| `0` | **Gateway** | Recebe tráfego externo/internet (Proxies, Load Balancers). |
| `1` | **Frontend / UI** | Servidores de assets estáticos. |
| `2` | **Backend Core** | APIs e camada de regras de negócios (Rust, C++, Java). |
| `3` | **Bancos de Dados** | Isolamento máximo (PostgreSQL, ScyllaDB, etc.). |
| `4` | **Mensageria** | Barramento de eventos (Kafka, RabbitMQ, etc.). |
| `5` | **Batch Processing** | Processos em background e workers genéricos. |
| `6` | **IA - Inferência** | Execução de modelos locais e workloads de GPU (C++/CUDA). |
| `7` | **IA - Agentes** | Serviços de IA e orquestração. |
| `8` | **Data Science** | Redes exclusivas para treinamento e pipelines de dados (Python). |
| `9` | **Cache / KV** | Armazenamento de alta velocidade (Redis, Memcached). |
| `a` | **Telemetria** | Observabilidade *read-only* (Prometheus, Logs). |
| `b` | **Management** | Acesso administrativo e *Control Plane* (SSH, Bastion, Kubernetes API). |
| `f` | **Outros** | Serviços não classificados (Requer revisão periódica). |

### N4: Cluster ID (Identificação do Nó/Zona)
Identifica zonas de disponibilidade do mesmo serviço.

| Hex | ID | Descrição |
| :--- | :--- | :--- |
| `0` | **Principal** | Cluster Primário / Zona A. |
| `1` | **Secundário** | Cluster Secundário / Zona B (Alta disponibilidade). |
| `f` | **Routing** | Tráfego interno de roteadores de interligação. |

## 3. Padrão do Interface ID (Últimos 64 bits)

Para manter a previsibilidade no gerenciamento Zero-Trust, o uso de SLAAC (configuração automática) ou extensões de privacidade é **proibido** em instâncias de servidores.

Os últimos 64 bits devem refletir o propósito do IP usando notação hexadecimal estática (*Vanity IPs*):

| Final | Uso | Exemplo de Aplicação |
| :--- | :--- | :--- |
| `::1` | **Gateway Padrão (Router)** | O roteador da respectiva sub-rede. |
| `::10` a `::99` | **Nós de Infraestrutura** | Instâncias estáticas, Master Nodes. Ex: Banco de Dados Nó 1 (`::10`). |
| `::a0` a `::ff` | **Serviços Anycast / VIPs** | IPs flutuantes (Keepalived, MetalLB). |
| `::xxxx` | **Dinâmico Seguro** | IPs entregues via DHCPv6 para containers/pods efêmeros. |
